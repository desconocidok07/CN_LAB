Welcome to the Caching Demo!

This page demonstrates HTTP caching using ETag and Last-Modified headers.

How it works:

  • First visit: Server sends full content with ETag and Last-Modified headers
  • Subsequent visits: Browser sends cached validators
  • If content unchanged: Server responds with 304 Not Modified
  • If content changed: Server sends new content with updated headers
